# Integer、Long缓存小数值对象

Integer、Long会缓存 -128~127之间的对象。如Interger类中有个静态内部类IntegerCache，其中的静态代码块初始化并缓存了-128~127之间的Integer对象。

jdk源码：

private static class IntegerCache {
    static final int low = -128;
    static final int high;
    static final Integer cache[];

    static {
        // high value may be configured by property
        int h = 127;
        String integerCacheHighPropValue =
            sun.misc.VM.getSavedProperty("java.lang.Integer.IntegerCache.high");
        if (integerCacheHighPropValue != null) {
            int i = parseInt(integerCacheHighPropValue);
            i = Math.max(i, 127);
            // Maximum array size is Integer.MAX_VALUE
            h = Math.min(i, Integer.MAX_VALUE - (-low));
        }
        high = h;

        cache = new Integer[(high - low) + 1];
        int j = low;
        for(int k = 0; k < cache.length; k++)
            cache[k] = new Integer(j++);
    }

    private IntegerCache() {}
}

测试代码：

@Test
public void testInterger() {
    Integer integer1 = new Integer(256);
    Integer integer2 = new Integer(256);
    Integer integer3 = new Integer(120);
    Integer integer4 = new Integer(120);
    System.out.println(integer1 == integer2);
    System.out.println(integer3 == integer4);
    System.out.println((Integer)(130) == Integer.valueOf(130));
    System.out.println(Integer.valueOf(130) == Integer.valueOf(130));
    System.out.println(Integer.valueOf(120) == Integer.valueOf(120));
    System.out.println((Integer)(120) == Integer.valueOf(120));
}

测试输出：

false
false
false
false
true
true

将上述的Interger换成Long 输出结果相同。

使用Integer.valueOf()、自动装箱和强制类型转换都会优先使用缓存对象。使用 new Integer()不会使用缓存对象。

# Throwable

  • Error：是系统中的错误，程序员是不能改变的和处理的，是在程序编译时出现的错误，只能通过修改程序才能修正。一般是指与虚拟机相关的问题，如系统崩溃，虚拟机错误，内存空间不足，方法调用栈溢等。对于这类错误的导致的应用程序中断，仅靠程序本身无法恢复和和预防，遇到这样的错误，建议让程序终止。
  • Exception：表示程序可以处理的异常，可以捕获且可能恢复。遇到这类异常，应该尽可能处理异常，使程序恢复运行，而不应该随意终止异常。

# Exception

Exception又分为两类:

  1. CheckedException：（编译时异常） 需要用try——catch显示的捕获，对于可恢复的异常使用CheckedException。
  2. UnCheckedException（RuntimeException）：（运行时异常）不需要捕获，对于程序错误（不可恢复）的异常使用RuntimeException。

常见的RuntimeException异常:

  1. illegalArgumentException：此异常表明向方法传递了一个不合法或不正确的参数。
  2. illegalStateException：在不合理或不正确时间内唤醒一方法时出现的异常信息。换句话说，即 Java 环境或 Java 应用不满足请求操作。
  3. NullpointerException：空指针异常（我目前遇见的最多的）
  4. IndexOutOfBoundsException：索引超出边界异常

常见的CheckedException异常

  1. 我们在编写程序过程中try——catch捕获到的一场都是CheckedException。
  2. io包中的IOExecption及其子类，都是CheckedException。

# supressedException

开发人员对异常处理的try-catch-finally语句块都比较熟悉。如果在try语句块中抛出了异常，在控制权转移到调用栈上一层代码之前，finally语句块中的语句也会执行。但是finally语句块在执行的过程中，也可能会抛出异常。如果finally语句块也抛出了异常，那么这个异常会往上传递，而之前try语句块中的那个异常就丢失了。

如下示例中如果关闭连接发生异常则建立连接和执行语句中的异常就丢失了。

try{
    建立连接执行sql
}catch(Exception e){
    logger...
}finally{
    关闭连接
}

在java7中为Throwable类增加addSuppressed方法。当一个异常被抛出的时候，可能有其他异常因为该异常而被抑制住，从而无法正常抛出。这时可以通过addSuppressed方法把这些被抑制的方法记录下来。被抑制的异常会出现在抛出的异常的堆栈信息中，也可以通过getSuppressed方法来获取这些异常。这样做的好处是不会丢失任何异常，方便开发人员进行调试。

Exception exception = null;
try{
    建立连接执行sql
}catch(Exception e1){
    exception = e1;
    logger...
}finally{
    try{
        关闭连接
    }catch(Exception e2){
        exception.addSuppressed(e2)
    }
}
if(exception != null) {
    throw exception
}

# Class

  • getFields()：获得某个类的所有的公共（public）的字段，包括父类中的字段。
  • getDeclaredFields()：获得某个类的所有声明的字段，即包括public、private和proteced，但是不包括父类的申明字段。

同样类似的还有getConstructors()和getDeclaredConstructors()、getMethods()和getDeclaredMethods()，这两者分别表示获取某个类的方法、构造函数。
