# 数据收集埋点方式

# 代码埋点

# 介绍

在需要埋点点位置植入埋点代码，将数据上报。

# 优点

  • 采集时机灵活，可以在任何位置、任何时机采集数据。
  • 采集数据格式灵活，可以自定义属性、自定义事件，能传递比较丰富的数据到服务端。

# 缺点

  • 埋点代价比较大，代码冗余，只能由技术人员实施；
  • 功能调整时，需要调整对于埋点；
  • 埋点策略更新需要应用升级(h5、小程序、快应用除外)。

# 可视化埋点

# 介绍

使用可视化交互手段代替写代码；在可视化平台上配置需要采集的控件、以及需要采集的信息。把功能代码和埋点配置资源分开，每次用户启动app的时候通过网络更新配置资源。

# 优点

  • 埋点无代码侵入；
  • 可由运营人员实施；
  • 可以实时更新。

# 缺点

  • 埋点位置比较局限，只能在页面上部分控件的特定事件处埋点，动态页面和动效控件无法埋点。

# 无埋点

# 说明

无埋点即全埋点，无差异的收集所有控件的操作数据。

# 优点

  • 开发人员工作量小，无需对业务标识进行唯一区分，由sdk自动进行生成，ID规则由sdk和产品进行约定；
  • 数据量全面，产品可按需进行分析；
  • 支持动态页面和局部动效的统计。

# 缺点

  • 数据量大，服务端压力大。
